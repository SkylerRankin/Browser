This browser implements the bare minimum HTML and CSS to view many simple webpages and most older webpages. The application's main pipeline to convert a URL to a visible site happens in about 3 steps: resource fetching, layout calculation, and rendering. These steps, particularly layout calculation, contain many smaller steps and explore lots of interesting HTML rendering engine problems, such as generating a DOM, resolving complex CSS precedence rules, and intelligent line splitting.

In the bookmarks above are some webpages that render pretty closely to what modern browser would provide. Given that this is largely an HTML renderer (no JavaScript engine), lots of modern sites won't render correctly.
